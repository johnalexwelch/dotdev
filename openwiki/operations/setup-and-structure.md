## Installation: Fresh Machine

### Prerequisites

- **macOS** (10.15+)
- **Git** (pre-installed)
- **Homebrew** (will be installed by `install.sh`)

### Install

```bash
# Clone the repository
git clone git@github-personal:johnalexwelch/dotdev.git ~/dotdev

# Run the bootstrap script
cd ~/dotdev
bash install.sh

# Install pre-commit hooks
pre-commit install

# Verify
zsh  # Switch to zsh
echo $SHELL  # Should print /bin/zsh
```

### What `install.sh` Does (In Order)

1. **`brew.sh`** — Installs Homebrew (if missing), runs `brew bundle --file=Brewfile` to install all formulae + casks
2. **`config-init.sh`** — Pre-creates `~/.config/ghostty`, `~/.config/lazygit`, `~/.config/zsh`, etc. to prevent Stow tree-folding
3. **`github.sh`** — Generates `~/.ssh/id_ed25519`, adds to ssh-agent + keychain, authenticates with `gh`
4. **`gh-extensions.sh`** — Installs GitHub CLI extensions
5. **Application symlinks** — Links `~/.config/arc` → `~/Library/Application Support/Arc`, etc. (Arc, Cursor, StreamDeck)
6. **macOS defaults** — Runs all scripts in `scripts/macos/`: system defaults, Finder, Dock, Spotlight, Terminal, Screen, input devices, permissions
7. **`stow`** — Materializes all `dotfiles/` symlinks into `$HOME` via `stow -d ~/dotdev -R -t $HOME dotfiles`
8. **`ai-setup.sh`** — Installs guardian, gbrain, and pi packages
9. **`herdr-setup.sh`** — Configures herdr integrations and plugins

### Dry Run

To preview without executing:

```bash
DRY_RUN=1 bash install.sh
```

Prints every command without running it.

### Idempotency

`install.sh` is **idempotent** — safe to re-run. It:

- Skips Homebrew install if already installed
- Overwrites config files (safe — they're tracked in dotfiles/)
- Skips already-stowed files
- Regenerates SSH keys only if missing

## GNU Stow: Symlink Management

Stow materializes `dotfiles/` into `$HOME`:

```bash
# Materialize all symlinks
stow -d ~/dotdev -R -t $HOME dotfiles

# Preview (dry-run)
stow -d ~/dotdev -R -t $HOME dotfiles --simulate

# Unstow (remove symlinks)
stow -d ~/dotdev -R -t $HOME dotfiles -D
```

**How it works**:

- `dotfiles/.zshrc` → `~/.zshrc`
- `dotfiles/.config/zsh/` → `~/.config/zsh/`
- `dotfiles/.claude/` → `~/.claude/`
- Directories are nested; files are symlinked individually

**config-init.sh safety**: Before stow runs, `config-init.sh` pre-creates all target directories (`~/.config/zsh`, `~/.config/nvim`, etc.). This prevents Stow from "folding" directories into single symlinks, which would break organization.

## Configuration Management

### Adding a New Tool

1. **Create config directory**:
   ```bash
   mkdir -p dotfiles/.config/my-tool
   ```

2. **Add config files**:
   ```bash
   cp ~/path/to/my-tool/config dotfiles/.config/my-tool/
   ```

3. **Add to config-init.sh** (if new top-level config dir):
   ```bash
   mkdir -p "$HOME/.config/my-tool"
   ```

4. **Stow and test**:
   ```bash
   cd ~/dotdev
   stow -d . -R -t $HOME dotfiles
   ```

### Updating Configuration

All changes to `dotfiles/` apply after stow:

```bash
# Edit a config file
vim dotfiles/.config/zsh/configs/aliases.zsh

# Regenerate symlinks (safe)
cd ~/dotdev && stow -d . -R -t $HOME dotfiles

# Reload shell (if needed)
source ~/.zshrc
```

### Environment Variables & Secrets

Secret values are **sourced at shell load time** from credential files:

```bash
# ~/.config/zsh/configs/env.zsh
source ~/.anthropic     # ANTHROPIC_API_KEY
source ~/.openai        # OPENAI_API_KEY
source ~/.slack         # SLACK_TOKEN, etc.
```

These files are:
- **Not tracked** in dotdev (listed in `.gitignore`)
- **Created manually** on each machine
- **Sourced at login** (dotfiles/.zshrc runs them)

To set up credentials on a fresh machine:

```bash
# Create credential files (not tracked by git)
cat > ~/.anthropic << EOF
export ANTHROPIC_API_KEY="sk-..."
EOF

cat > ~/.openai << EOF
export OPENAI_API_KEY="sk-..."
EOF
```

## Maintenance

### Updating Homebrew Packages

Edit `Brewfile`:

```bash
vim Brewfile
```

Then install/update:

```bash
brew bundle
```

### Updating Skills

All skills live in `dotfiles/.config/agents/skills/`:

```bash
# Edit a skill
vim dotfiles/.config/agents/skills/my-skill/SKILL.md

# Regenerate skills index
dotfiles/.config/agents/skills/_docs/skills-index.sh --write

# Commit
git add dotfiles/.config/agents/skills/
git commit -m "skill(my-skill): update process"
```

### Running Tests

Test suite is in `test/`:

```bash
cd ~/dotdev
bash test/run-tests.sh
```

Tests cover:
- Shell syntax (bash -n)
- Pre-commit hooks
- Stow symlink safety

### Updating Documentation

User-facing docs are in `docs/`:

```bash
# Edit a doc
vim docs/SHELL.md

# Linting
markdownlint docs/SHELL.md

# Commit
git commit -m "docs(shell): clarify alias behavior"
```

OpenWiki-generated docs are in `/openwiki/`:

```bash
# OpenWiki docs are auto-generated; don't hand-edit
# Instead, update source files and re-run OpenWiki

openwiki --update "Add new section"
```

## Troubleshooting

### Stow Creates Symlinks But Files Don't Appear in $HOME

**Cause**: `config-init.sh` didn't run or didn't create target directories.

**Fix**:
```bash
bash scripts/config-init.sh
stow -d ~/dotdev -R -t $HOME dotfiles
```

### Homebrew Packages Missing

**Cause**: `Brewfile` changed but `brew bundle` didn't run.

**Fix**:
```bash
brew bundle
```

### Pre-commit Hooks Not Running

**Cause**: Not installed or incorrect config.

**Fix**:
```bash
cd ~/dotdev
pre-commit install
pre-commit run --all-files  # Test hooks
```

### Guardian Compile Failure

**Cause**: TypeScript error in `~/.claude/hooks/guardian/*.ts`.

**Fix**:
```bash
cd ~/.claude/hooks
npx tsc  # Check for errors
# Edit the TypeScript file to fix errors
# PostToolUse will auto-recompile on save
```

### Shell Config Not Loading

**Cause**: Symlink broken or `.zshrc` not sourced.

**Fix**:
```bash
# Check if ~/.zshrc is a symlink
ls -la ~/.zshrc

# If not, re-stow
cd ~/dotdev && stow -d . -R -t $HOME dotfiles

# Reload
source ~/.zshrc
```

## Key Files for Operators

| File | Purpose | Change frequency |
|------|---------|---|
| `Brewfile` | Package dependencies | Often; re-run `brew bundle` |
| `dotfiles/.config/zsh/` | Shell configuration | Often; hotload with `source` |
| `dotfiles/.config/agents/skills/*/SKILL.md` | Skill specifications | Often (via workflow-review) |
| `docs/decision-log.md` | Design decisions | Append-only (never edit prior entries) |
| `scripts/install.sh` | Bootstrap | Rarely (test thoroughly) |
| `.pre-commit-config.yaml` | Pre-commit hooks | Rarely (test thoroughly) |
| `AI_ENVIRONMENT.md` | System overview | Rarely (keep in sync with /openwiki) |
