- **Task 2 — Subagent types adapted.** Production skill uses `general-purpose` for three roles: `[auto]` cluster dispatch (Step 4), post-batch scope verification (Step 5), phase-level verification against the plan's Verification text (Step 7). `Explore` type not used — this skill mutates; Explore is read-only.

- **Task 3 — Vocabulary adapted.** Outcome files: `docs/executions/.phase-runs/<date>[-<plan-slug>]-phase-<N>.md`. Commit messages: `phase-<N>: <Goal> (addresses <FIND-NN list>)`. Branch naming: `refactor/phase-<N>-<phase-slug>` per plan §11. No Desktop-specific paths or ticket slugs remain.

- **Task 4 — Full preflight.** Step 0 resolves `plan_path` (default newest), derives `plan_slug` from filename stem (strip date prefix + `-design` suffix), verifies phase number, checks for clean working tree (aborts on dirty), detects existing outcome file (aborts without `resume=true`), checks ID scheme (FIND-NN/GAP-NN/other), creates docs dirs. NEW-05 addressed here: `plan_slug` input + default-derivation resolves the two-plans-same-phase-N collision the pilot surfaced.

- **Task 5 — Branch creation.** Step 3 creates `refactor/phase-<N>-<phase-slug>` off current HEAD. Current HEAD = `main` for first phase, prior phase's HEAD for subsequent. Aborts on branch-already-exists. Recorded in outcome file `## Commits` with starting short-hash.

- **Task 6 — Parallel cluster dispatch with scope isolation.** Step 2 partitions tasks into clusters by surface area; Step 4 dispatches one `general-purpose` Agent per cluster with an explicit disjoint file-scope grant in the prompt. Parallel when scopes are disjoint (one message, multiple Agent calls, per `/repo-audit` pattern). Step 5 post-batch verification subagent diffs `git status --porcelain` against granted scopes — **this diff runs in its own subagent rather than inside the mutating cluster subagents**, which directly addresses NEW-04 (the bare-`ls` truncation Phase 1 surfaced; orchestrator-side verification can't be spoofed by cluster subagents whose output might be truncated). Cluster subagent prompts also include a secondary instruction to prefer absolute-path invocations for evidence-bearing commands.

- **Task 7 — `[human]` tasks surfaced, never executed.** Step 6 writes to `## Pending human`, prints to chat, halts the phase. No execution under any condition, including `dry_run`.

- **Task 8 — Verification subagent.** Step 7 dispatches a separate `general-purpose` subagent with the phase's Verification text. Reports PASS/FAIL/UNVERIFIED per claim with evidence. On FAIL: no commit, halt, quote Rollback text to user as reference. On UNVERIFIED load-bearing: halt for user call.

- **Task 9 — Commit + auto-proceed.** Step 8 stages files scoped to the union of granted scopes (never wider — belt-and-suspenders after scope verification), commits with `phase-<N>: <Goal> (addresses <IDs>)` schema. Step 9 auto-proceeds on clean verification + no pending `[human]` + no scope violations + no failed tasks — recurses into Step 0 with `phase = N+1`. Halts on any of: verification fail, `[human]` gate, scope violation, failed task, `auto_proceed=false`, phase N+1 not in plan, phase N+1 preflight error. Halt leaves phase branch checked out for user resolution.

- **Task 10 — Outcome file schema.** Step 10 defines the schema: `## Executed` (with cluster-N annotation), `## Pending human`, `## Verification` (overall + per-claim), `## Rollback (reference)`, `## Commits` (hashes + messages + changed files), `## Scope violations`, `## Plan parse warnings`, `## Follow-ups` (NEW-NN candidates), `## Chain state` (full auto-proceed chain status).

- **Task 11 — Error Handling.** Table covers 16 failure modes: dirty tree, missing plan, missing phase header, lacking FIND-NN, existing outcome file, empty Tasks, branch collision, subagent task fail, scope violation, human gate, verification fail, verification unverified load-bearing, dry_run, end-of-plan, mid-chain preflight fail, resume with no prior outcome.

- **Task 12 — Tuning notes.** 10 bullets covering: parallel vs. serial cluster dispatch, scope granularity, **evidence commands and subagent output truncation (NEW-04 folded in explicitly)**, when to use `/setup-worktree`, when to skip auto-proceed, `dry_run` recommendation, **outcome-filename plan_slug schema (NEW-05 folded in explicitly)**, commit-message schema as load-bearing contract, first-phase parent, port-source provenance.

- **Task 13 — Example Invocation.** Two fenced examples: clean auto-proceed chain through several phases (hits `[human]` at phase 3), and a single-phase invocation that hits a `[human]` gate after verification PASS. Both show the exact chat summary format from Step 11.
