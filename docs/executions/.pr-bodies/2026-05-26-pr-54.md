## Summary

Adds a "Non-default-branch close fallback" section to `dotfiles/.claude/skills/reconcile-issues/SKILL.md` that auto-closes `Closes/Fixes/Resolves`-referenced issues whenever a merged PR's base branch isn't the repository's default branch — working around GitHub's documented auto-close limitation (native keywords only fire on default-branch merges).

The fallback runs inside `workflow-finalize` Step 4 with a narrowly-scoped approval-bypass: only the exact trigger (merged + non-default base + open issue + valid `Closes/Fixes/Resolves #N` ref in PR body) auto-closes. All other close paths in the skill remain approval-gated.

Also adds `docs/decision-log.md` capturing the 7 grill decisions (DL-0001 … DL-0007) that produced this design — establishing the precedent decision-record format for future `workflow-feature` flows in this repo.
